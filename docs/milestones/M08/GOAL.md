# M08 — Web-Only AI Story Understanding Validation

## Objective

Make the loopback browser the sole supported product surface and prove that evidence-grounded
GPT-5.6 Luna organization turns deterministic Ren'Py structure into a genuinely readable story map
on controlled synthetic, small-real, recovered-source, and bounded walkthrough-backed examples.

## Deliverables

- Remove the legacy standalone PySide6/QGraphicsView graph application and entry point while
  retaining only a narrowly scoped local file/folder picker if the browser workflow requires it.
- Define AI Story Map as the required human-readable stage and Technical Structure as its factual
  verification/fallback view.
- Add a reproducible evaluation manifest, rubric, runner, reports, and browser comparison flow.
- Validate scene boundaries, meaningful events, titles, route meaning, character developments,
  detours, persistent routes, loops, and endings with evidence-linked Luna output.
- Preserve exact deterministic authority, explicit per-run cloud consent, recovered-source
  acknowledgement, bounded parallelism, checkpoints, cache, cancellation/resume, and draft review.
- Show AI coverage/fallback, scope status, calls, tokens, time, ETA, provider identity, and cache
  reuse in the browser.
- Complete Windows verification, independent review, milestone report, native infographic, and one
  unmerged PR.

## Acceptance criteria

1. The wheel and documented launch path expose only the local browser product; no standalone graph
   GUI entry point or QGraphicsView product code remains.
2. The generated complex fixture, small real `.rpy`, and small recovered `.rpyc` complete the
   baseline/AI/comparison workflow with stable deterministic authority and zero-call replay.
3. Bounded MsDenvers scopes are compared against curated walkthrough expectations without making
   the walkthrough a product dependency or committing copyrighted content.
4. Every AI interpretation is supported by existing evidence; no AI output changes factual
   connections, conditions, effects, merges, loops, routes, or endings.
5. The accepted AI Story Map is the default after organization and is measurably more readable than
   Technical Structure under the checked-in rubric.
6. Opening, reopening, and navigating invoke no provider. Live story transmission requires exact
   consent and records GPT-5.6 Luna, High reasoning, fast disabled, usage, timing, and hashes.
7. Cancel/resume, partial validated results, cache replay, review/apply/discard, coverage, fallback,
   and progress reporting work through the browser.
8. Windows CPython 3.12 pytest, Ruff, strict mypy, `pip check`, diff/wheel checks, live Chrome at
   100%/200%, end-to-end evaluation, and independent review pass with no unresolved P0-P2.
9. Completion report and native infographic exist; the single M08 PR remains unmerged.

## Exclusions

- No LM Studio, hosted deployment, installer, standalone executable, macOS compatibility,
  unrestricted full-game cloud rerun, game editing, game patching, or analyzer rewrite.
