# M05 Completion Report

## Outcome

M05 delivers the final approved product milestone: a Windows Story Explorer that keeps the M01-M04
deterministic graph authoritative while optionally organizing its evidence into reviewable arcs and
events through the user's ChatGPT/Codex login.

The accepted cloud path is locked to GPT-5.6 Luna with High reasoning and fast mode disabled. AI
cannot supply graph edges, requirements, effects, route destinations, or source locations. Every
successful run produces a draft requiring review; deterministic organization remains usable when
AI is disabled, rejected, unavailable, or incomplete.

Implementation was merged by the user through PR #7 at `2df4d67`. The final real-script evidence,
completion documentation, and infographic use a follow-up closeout branch because PR #7 was merged
before that last authorized smoke run.

## Delivered behavior

- Transactional schema-v4 organization runs, chunks, cache records, drafts, accepted arcs/events,
  locally derived quotient edges, claims/evidence, and durable edits.
- Consent-before-cloud organization through an isolated, sterile, read-only, ephemeral Codex CLI
  process with strict structured-output schemas and sanitized failures.
- Fail-closed GPT-5.6 Luna/High enforcement at provider and workflow boundaries, with fast mode,
  shell, web, plugins, apps, browser, MCP, and multi-agent features disabled.
- Bounded scene-first event grouping, reconciliation, arc grouping, exact-ID validation, one repair
  retry, and deterministic fallback for rejected or ungrouped material.
- Stable deterministic layout with SCC handling, branch lanes, semantic styles, and a 240-rendered-
  item hard cap.
- Adaptive Windows welcome screen and three-pane Story Explorer with arc, event, and exact-evidence
  levels; search, filters, keyboard focus, 200% zoom, inspector tabs, and contextual corrections.
- Review-before-apply plus rename, split, merge, move, hide, pin, approve, reject, discard, and
  reopen persistence.
- Repository-owned complex branching fixture with 27 choices, 26 gates, 88 effects, four endings,
  loop, merge, calls/returns, and one unresolved dynamic jump.

## Limitations and deferred work

- Local LM Studio organization is intentionally dormant and unvalidated in the M05 interface.
- Full canonical-game cloud organization was not attempted; deterministic/canonical UI evidence
  from M03-M04 remains authoritative.
- Database compaction and the historically large canonical project are deferred.
- AI summaries remain interpretations and can be imperfect; exact deterministic evidence is always
  available, and the user must review drafts before applying them.
- The small real script is a language/tutorial setup rather than a complex narrative. It proves the
  real `.rpy` path and fallback behavior; the synthetic fixture provides multi-route acceptance.
- Qt's offscreen test plugin exposed no system fonts in the final smoke environment. The normal
  Windows Qt plugin rendered correctly and is the release authority.
- Packaging, installer, public release, macOS, chatbot Q&A, and automatic ending finding remain out
  of scope. No M06 is implied.

## Safety and archive status

No embedded Ren'Py or game Python was executed. No source was modified. No credentials, project
databases, cache payloads, or extracted game content are committed. The canonical `scripts.rpa`
archive was not accessed during the M05 AI runs, so its established M04 fingerprint remains the
latest canonical evidence.

## Product completion

The planned M01-M05 roadmap is complete. A user can statically analyze a Ren'Py source, inspect the
deterministic three-level map, optionally organize it into readable arcs/events with explicit cloud
consent, review or reject every AI group, compare accepted and technical views, inspect exact
physical-line evidence, apply corrections, and reopen without repeating unchanged provider work.
